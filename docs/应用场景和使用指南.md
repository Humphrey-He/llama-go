# llama-go 应用场景和使用指南

## 应用场景

### 1. 本地 AI 助手服务
- **场景**: 企业内部知识库问答、文档分析
- **优势**: 数据不出网、低延迟、可离线运行
- **用例**: 员工培训、内部文档检索、代码审查助手

### 2. 边缘计算推理
- **场景**: IoT 设备、移动应用的本地推理
- **优势**: 减少网络往返、降低云成本
- **用例**: 智能家居控制、移动端 AI 功能

### 3. 开发工具集成
- **场景**: IDE 插件、CLI 工具、代码生成
- **优势**: 快速集成、OpenAI 兼容 API
- **用例**: 代码补全、文档生成、测试用例生成

### 4. 多租户 SaaS 后端
- **场景**: 为多个应用提供统一推理服务
- **优势**: 会话管理、速率限制、模型路由
- **用例**: 聊天应用、内容生成平台

## 快速开始

### 1. 启动服务
```bash
docker-compose up -d
```

### 2. 查看可用模型
```bash
curl http://localhost:8080/v1/models
```

### 3. 发送聊天请求
```bash
curl -X POST http://localhost:8080/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "tinyllama-chat",
    "messages": [
      {"role": "user", "content": "你好，请介绍一下自己"}
    ],
    "temperature": 0.7,
    "max_tokens": 512
  }'
```

### 4. 流式响应
```bash
curl -X POST http://localhost:8080/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "tinyllama-chat",
    "messages": [
      {"role": "user", "content": "写一个 Python 快速排序"}
    ],
    "stream": true,
    "max_tokens": 1024
  }'
```

## 使用用例

### 用例 1: 知识库问答系统

**场景**: 企业内部文档检索

**实现步骤**:
1. 用户提问
2. 系统检索相关文档
3. 将文档作为上下文发送给 llama-go
4. 返回基于文档的答案

**示例代码**:
```bash
# 带上下文的问答
curl -X POST http://localhost:8080/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "tinyllama-chat",
    "messages": [
      {
        "role": "system",
        "content": "你是一个企业知识库助手。根据以下文档回答问题。\n\n文档：公司成立于2020年，主要业务是AI解决方案。"
      },
      {
        "role": "user",
        "content": "公司什么时候成立的？"
      }
    ],
    "temperature": 0.3,
    "max_tokens": 256
  }'
```

### 用例 2: 代码生成助手

**场景**: IDE 集成的代码补全

**实现步骤**:
1. 用户输入代码片段
2. 发送给 llama-go 请求补全
3. 流式返回生成的代码
4. IDE 实时显示建议

**示例代码**:
```bash
# 代码生成请求
curl -X POST http://localhost:8080/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "tinyllama-chat",
    "messages": [
      {
        "role": "system",
        "content": "你是一个代码生成助手。生成高质量的 Go 代码。"
      },
      {
        "role": "user",
        "content": "写一个 HTTP GET 请求的函数"
      }
    ],
    "stream": true,
    "max_tokens": 512
  }'
```

### 用例 3: 多轮对话会话

**场景**: 聊天应用

**实现步骤**:
1. 创建会话 ID
2. 保持会话上下文
3. 多轮对话自动管理历史
4. 清理过期会话

**示例代码**:
```bash
# 第一轮对话
curl -X POST http://localhost:8080/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "tinyllama-chat",
    "messages": [
      {"role": "user", "content": "我叫张三"}
    ],
    "session_id": "user-123-session-1"
  }'

# 第二轮对话（会话保持）
curl -X POST http://localhost:8080/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "tinyllama-chat",
    "messages": [
      {"role": "user", "content": "我叫什么名字？"}
    ],
    "session_id": "user-123-session-1"
  }'
```

### 用例 4: 内容生成平台

**场景**: 自动生成文章、摘要、标题

**实现步骤**:
1. 接收内容生成请求
2. 构建适当的 prompt
3. 调用 llama-go API
4. 返回生成内容

**示例代码**:
```bash
# 文章摘要生成
curl -X POST http://localhost:8080/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "tinyllama-chat",
    "messages": [
      {
        "role": "system",
        "content": "你是一个专业的文章摘要生成器。生成简洁、准确的摘要。"
      },
      {
        "role": "user",
        "content": "请为以下文章生成摘要：[长文章内容]"
      }
    ],
    "temperature": 0.5,
    "max_tokens": 256
  }'
```

## API 端点参考

### 健康检查
- `GET /healthz` - 进程存活检查
- `GET /readyz` - 服务就绪检查

### 模型管理
- `GET /v1/models` - 列出所有可用模型

### 聊天接口
- `POST /v1/chat/completions` - 发送聊天请求（支持流式）

### 会话管理
- `POST /clear-session` - 清理会话

---

## 性能优化建议

1. **批量请求**: 使用会话 ID 复用连接
2. **流式响应**: 对长文本使用 `stream: true`
3. **温度调整**:
   - 创意任务: `temperature: 0.7-0.9`
   - 精确任务: `temperature: 0.1-0.3`
4. **Token 限制**: 根据场景设置合理的 `max_tokens`
