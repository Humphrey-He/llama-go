# llama-go GGUF 模型部署指南

## 快速答案

**可以直接下载 GGUF 文件放到项目里运行吗?**

✅ **可以**，但需要通过后端服务（llama.cpp 或 vLLM）来加载和推理。

---

## 推荐架构

```
用户请求
    ↓
llama-go (Go API) :8080
    ↓
Python/llama.cpp 后端 :8000
    ↓
GGUF 模型文件
```

---

## 部署步骤

### 1. 下载模型

```bash
# 创建模型目录
mkdir -p models

# 方式1: 使用 huggingface-hub
pip install huggingface-hub
huggingface-cli download TheBloke/Dolphin-3.0-Llama-3.1-8B-GGUF \
  dolphin-3.0-llama-3.1-8b.Q4_K_M.gguf \
  --local-dir ./models

# 方式2: 手动下载
# 访问 https://huggingface.co/TheBloke/Dolphin-3.0-Llama-3.1-8B-GGUF
# 下载 Q4_K_M.gguf 文件到 ./models 目录
```

### 2. 启动后端服务

**选项 A: llama.cpp (推荐轻量级)**

```bash
# 安装 llama.cpp
git clone https://github.com/ggerganov/llama.cpp
cd llama.cpp
make

# 启动服务
./server -m ../models/dolphin-3.0-llama-3.1-8b.Q4_K_M.gguf \
  --port 8000 \
  -n 256 \
  -c 2048
```

**选项 B: vLLM (功能完整)**

```bash
pip install vllm

python -m vllm.entrypoints.openai.api_server \
  --model ./models/dolphin-3.0-llama-3.1-8b.Q4_K_M.gguf \
  --port 8000
```

### 3. 启动 llama-go

```bash
# 终端2
go run ./cmd/inference/main.go
```

### 4. 测试

```bash
curl -X POST http://localhost:8080/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "tinyllama-chat",
    "messages": [{"role": "user", "content": "Hello"}],
    "stream": false
  }'
```

## 常见问题

**Q: 需要 GPU 吗?**
A: 不必需，CPU 也能运行，但 GPU 会加速 10-50 倍

**Q: 如何切换模型?**
A: 修改后端启动命令中的模型路径，重启后端

**Q: 模型文件很大怎么办?**
A: 使用量化版本（Q4_K_M），约 5GB，质量损失很小
