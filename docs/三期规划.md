# llama-go 第三期开发规划（v0.2）

## 一、版本目标（Version Goal）

将 llama-go 从「原型工程」升级为：

> ✅ **可部署 / 可观测 / 可扩展 / 标准化 API 的在线推理服务**

核心定位：

* Go：控制面 + 服务编排
* 推理：接入成熟后端（vLLM / llama.cpp）
* 对外：提供 OpenAI 风格 API
* 支撑：多轮对话、流式返回、监控、限流

---

## 二、OKR 设计

### 🎯 O1：构建可上线的标准 LLM 服务（核心目标）

**KR1**

**KR3**

## 四、核心模块设计

### 1️⃣ API 层（internal/api）

职责：

* HTTP 路由
* 参数校验
* 响应格式统一

---

### 2️⃣ Service 层（internal/service）

职责：

* Prompt 拼接
* 多轮上下文管理
* token usage 计算
* backend 调度

---

### 3️⃣ Backend 层（internal/backend）

统一接口：

```go
type LLMBackend interface {
    Chat(ctx context.Context, req ChatRequest) (*ChatResponse, error)
    StreamChat(ctx context.Context, req ChatRequest) (<-chan StreamChunk, error)
    Health(ctx context.Context) error
}
```

实现：

* VLLMBackend
* LlamaCppBackend

---

### 4️⃣ Session 层（internal/session）

功能：

* 存储 messages
* 支持 TTL
* 支持最大上下文长度裁剪

---

### 5️⃣ Observability（internal/observability）

* metrics（Prometheus）
* logging（结构化日志）
* trace（可选）

---

## 五、接口定义（API Draft）

### 1. Chat Completions

#### POST /v1/chat/completions

```json
{
  "model": "llama-3-8b-instruct",
  "messages": [
    {"role": "system", "content": "你是一个助手"},
    {"role": "user", "content": "解释KV Cache"}
  ],
  "temperature": 0.7,
  "top_p": 0.9,
  "max_tokens": 512,
  "stream": true,
  "session_id": "user_123"
}
```

---

### Response（非流式）

```json
{
  "id": "req-xxx",
  "object": "chat.completion",
  "choices": [
    {
      "message": {
        "role": "assistant",
        "content": "..."
      },
      "finish_reason": "stop"
    }
  ],
  "usage": {
    "prompt_tokens": 100,
    "completion_tokens": 50,
    "total_tokens": 150
  }
}
```

---

### Response（流式 SSE）

```
data: { "delta": "hello" }
data: { "delta": " world" }
data: [DONE]
```

---

### 2. Clear Session

```
POST /v1/sessions/{id}/clear
```

---

### 3. 健康检查

```
GET /healthz
GET /readyz
```

---

### 4. Metrics

```
GET /metrics
```
